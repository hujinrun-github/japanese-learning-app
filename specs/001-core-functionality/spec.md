# 功能规范：日语学习应用 (Japanese Learning App)

## 产品定位与设计原则

### 目标用户画像

| 属性 | 描述 |
|---|---|
| 人群 | 有全职工作、利用碎片时间学日语的职场人士 |
| 日语水平 | 零基础 (N5 入门) 到中高级 (N1)，全范围覆盖 |
| 母语背景 | 以中文母语者为主，有一定汉字基础 |
| 痛点 | 时间碎片化、缺乏口语练习机会、学习进度难以坚持 |
| 学习场景 | 通勤（3~5 分钟快速复习）+ 午休/晚间（15~30 分钟深度学习）|

### 碎片时间设计原则

> **每一个最小学习单元，必须能在 5 分钟内独立完成，且支持随时中断、下次无缝续学。**

- 单词卡片：每张翻阅 < 15 秒
- 语法点：讲解 + 快速检验 < 5 分钟/条
- 跟读段落：每段音频 < 2 分钟
- 造句练习：每道题 < 3 分钟
- 所有模块均显示"今日进度"，方便用户感知学习量

### 平台优先级

| 阶段 | 平台 | 说明 |
|---|---|---|
| V1~V4 | **Web 浏览器** | 桌面 + 移动响应式，优先交付 |
| V5+ | **iOS 原生 App** | 后续迭代，复用后端 API |

| 版本 | 核心功能 | 优先级 |
|---|---|---|

## 用户故事与验收场景

### 用户故事 1 — 单词记忆与间隔重复（优先级：P1）

**场景描述**

学习者希望用科学方法记忆日语单词。打开单词记忆模块后，看到今日待复习的单词卡片队列。每张卡片正面显示日语单词（含假名注音），背面显示中文释义和例句。用户对记忆程度评分（容易 / 一般 / 困难），系统据此自动安排下次复习时间（SM-2 算法）。

**设计要点（碎片时间适配）**
- 每次最少可复习 1 张卡片，最多 20 张为一轮
- 卡片队列按优先级排序：到期复习 → 新词学习
- 中断后下次从断点续学

**验收场景**

| # | Given（前提）| When（操作）| Then（预期结果）|
|---|---|---|---|
| 1.1 | 用户首次进入单词记忆模块 | 打开模块 | 展示词库中第一批新学单词（10~20 个），并显示今日计划总数 |
| 1.2 | 用户正在复习一张卡片 | 翻开卡片后选择「困难」| 该单词在 **1 天内**重新出现 |
| 1.3 | 用户正在复习一张卡片 | 翻开卡片后选择「容易」| 该单词在 **7 天后**重新出现 |
| 1.4 | 用户今日复习已全部完成 | 队列为空时 | 显示今日学习总结（复习数量、正确率、连续学习天数），并提供「提前学习新词」选项 |
| 1.5 | 用户查看学习统计 | 进入统计页面 | 显示每个单词的掌握程度曲线和历史复习记录 |
| 1.6 | 用户在课文学习中标记了生词 | 点击「加入单词本」| 该词出现在单词记忆队列中，下次复习时可见 |
| 1.7 | 用户在通勤途中复习（网络不稳定）| 离线状态下打开模块 | 已缓存的单词卡片正常显示，评分结果本地暂存，恢复网络后自动同步 |

### 用户故事 2 — 语法学习（优先级：P2）

**场景描述**

学习者希望系统地学习日语语法。进入语法学习模块后，按 JLPT 等级（N5~N1）浏览语法点列表。点击一个语法点后，先阅读讲解和例句，再完成 2~3 道快速检验题（填空 / 选择），最后将该语法点加入间隔重复复习队列。

**设计要点（混合学习形式）**
- 每个语法点：讲解 → 例句 → 快速检验 → 加入复习
- 检验题答错后：立即展示详细解析，不直接进入下一题
- 语法点可标记「已掌握 / 学习中 / 未学」三种状态

**验收场景**

| # | Given（前提）| When（操作）| Then（预期结果）|
|---|---|---|---|
| 2.1 | 用户在语法列表页 | 按 N4 等级筛选 | 仅显示 N4 语法点，每条显示语法名称、简介、掌握状态 |
| 2.2 | 用户打开一个语法点「〜てもいい」| 进入详情页 | 显示：名称、意思（「可以〜」）、接续方式、至少 3 条例句（日语+中文翻译） |
| 2.3 | 用户完成语法讲解阅读 | 点击「开始检验」| 出现 2~3 道填空/选择题，题目使用该语法点造句，用户填写缺失部分 |
| 2.4 | 用户答错检验题 | 提交错误答案 | 立即显示正确答案和详细解析，不跳过，需用户确认后才进入下一题 |
| 2.5 | 用户完成所有检验题 | 检验完成 | 显示本次得分，提供「加入复习队列」按钮；加入后，该语法点参与间隔重复调度 |
| 2.6 | 用户在语法例句中看到生词 | 点击例句中的单词 | 弹出释义卡片（读音、词性、释义），并可一键加入单词本 |
| 2.7 | 管理员在后台操作 | 新增/编辑语法点内容 | 前台用户刷新后可见最新内容，不需要重新发布应用 |

### 用户故事 3 — 课文学习（优先级：P3）

**场景描述**

学习者希望通过阅读真实语境的日语课文提升综合能力。课文按 JLPT 等级分级，每篇课文的汉字上方标注假名（振り仮名），可点击单词查看释义，支持按句播放音频并高亮同步，可切换显示中文翻译。

**验收场景**

| # | Given（前提）| When（操作）| Then（预期结果）|
|---|---|---|---|
| 3.1 | 用户在课文列表 | 按 N3 筛选 | 仅显示 N3 级课文，标注标题、字数、主题标签 |
| 3.2 | 用户打开一篇课文 | 页面加载完成 | 日语文本正确渲染振り仮名，汉字与假名对齐，**加载时间 ≤ 3 秒** |
| 3.3 | 用户点击课文中的单词 | 单击任意单词 | 弹出释义卡片：读音、词性、中文释义、本文含义 |
| 3.4 | 用户点击句子旁的播放按钮 | 音频播放 | 当前朗读句子高亮显示，播放完毕后高亮消失 |
| 3.5 | 用户想看翻译 | 点击「显示翻译」| 切换为逐句中文对照模式，再次点击还原 |
| 3.6 | 用户完成课文阅读 | 点击「加入单词本」| 课文中所有标记生词自动添加到单词记忆队列 |
| 3.7 | 音频文件加载失败 | 点击播放 | 显示「加载失败，点击重试」按钮；文本内容不受影响，仍可正常阅读 |

### 用户故事 4 — 口语练习（优先级：P3）

**场景描述**

口语练习模块包含两个子模式，满足不同水平用户需求：

- **影子跟读（入门/进阶）**：跟着母语者原音模仿，系统对比录音给出相似度得分
- **自由朗读（进阶/挑战）**：无原音引导，用户自主朗读文章段落，系统评分并标注需注意的发音

**验收场景 — 影子跟读**

| # | Given（前提）| When（操作）| Then（预期结果）|
|---|---|---|---|
| 4.1 | 用户进入影子跟读 | 选择一段音频材料 | 显示对应文本（含假名）、音频进度条、速度控制（0.5x / 0.75x / 1x / 1.25x / 1.5x）|
| 4.2 | 用户调整速度至 0.75x | 点击播放 | 语调自然不失真，文本逐句同步高亮 |
| 4.3 | 用户点击录音跟读 | 录音结束 | 可回放自己的录音；系统显示整体相似度得分（0~100 分）；**反馈时间 ≤ 5 秒** |
| 4.4 | 用户查看历史记录 | 进入练习历史 | 显示每次练习的得分趋势折线图，以及所用材料名称和日期 |
| 4.5 | 用户麦克风权限被拒绝 | 进入跟读模式 | 提示授权说明；用户可在无录音情况下继续使用「听力跟读」（只听不录）|

**验收场景 — 自由朗读**

| # | Given（前提）| When（操作）| Then（预期结果）|
|---|---|---|---|
| 4.6 | 用户进入自由朗读 | 选择一篇课文段落 | 只显示文本（含假名），不播放原音 |
| 4.7 | 用户完成朗读录音 | 点击结束录音 | 系统给出整体发音得分，并标注哪些词/句发音需注意（高亮显示）；**反馈时间 ≤ 5 秒** |
| 4.8 | 用户想对比原音 | 点击「播放原音」| 原音播放，方便用户自我对比改进 |

### 用户故事 5 — 写作练习（优先级：P4）

**场景描述**

写作练习分两个子模式，按用户水平自动推荐：

- **输入练习（N5~N4）**：给出平假名/片假名或汉字提示，用户用日语输入法打出正确内容
- **造句练习（N4~N1）**：给出语法点和中文提示，用户用日语造句，AI 批改语法和用词

**验收场景 — 输入练习**

| # | Given（前提）| When（操作）| Then（预期结果）|
|---|---|---|---|
| 5.1 | N5 用户进入输入练习 | 打开模块 | 显示假名输入练习：展示一个假名，用户用键盘输入罗马字（自动转换为假名） |
| 5.2 | 用户输入正确 | 按下确认 | 高亮绿色提示「正确」，立即显示下一题 |
| 5.3 | 用户输入错误 | 按下确认 | 高亮红色显示正确答案，提示需再练习，2 秒后进入下一题 |
| 5.4 | N4+ 用户进入汉字练习 | 打开模块 | 显示单词的假名读音，用户需输入对应汉字形式（如：とうきょう → 東京）|

**验收场景 — 造句练习**

| # | Given（前提）| When（操作）| Then（预期结果）|
|---|---|---|---|
| 5.5 | 用户进入造句练习 | 打开模块 | 显示每日 3~5 道造句题：指定语法点 + 中文提示（如：「用〜てもいい 造句，表达"你可以坐这里"」）|
| 5.6 | 用户完成造句并提交 | 点击「提交」| AI 在 **10 秒内**给出反馈：语法正确性、用词准确性、改进建议；同时显示参考答案 |
| 5.7 | 用户收到 AI 反馈 | 查看反馈详情 | 详细说明哪里有问题（如「助词应为に而非を」），并给出修改后的完整句子 |
| 5.8 | 用户完成当日所有造句题 | 全部提交 | 显示今日写作总结：完成题数、平均得分、最常犯的错误类型 |

### 用户故事 6 — 用户账户与学习进度（优先级：P5）

**场景描述**

用户注册账号后，所有学习记录（单词掌握度、语法进度、课文完成情况、口语/写作历史）自动同步到服务器，支持多设备无缝继续学习，并提供完整的学习统计看板。

**验收场景**

| # | Given（前提）| When（操作）| Then（预期结果）|
|---|---|---|---|
| 6.1 | 新用户 | 完成注册 | 系统创建账号，初始化默认词库（N5 词库）和学习计划，跳转至首页引导 |
| 6.2 | 已登录用户 | 手机端完成单词复习 | Web 端登录后可看到相同的复习记录和单词掌握状态 |
| 6.3 | 用户连续学习 7 天 | 查看个人主页 | 显示连续打卡天数、总学习时长、各模块使用频率统计 |
| 6.4 | 用户未登录（游客模式）| 使用学习功能 | 可正常使用所有核心学习功能，数据存本地；提示「登录以同步进度」 |
| 6.5 | 用户同时在两台设备学习 | 数据冲突 | 以**最后更新时间戳**为准合并数据，不丢失任何学习记录 |

### 用户故事 7 — 练习总结（优先级：P2，各模块均适用）

**场景描述**

每次学习会话结束后，用户希望看到一份简洁的「练习总结」，清晰呈现本次练习做得好的地方和需要改进的地方，帮助用户建立学习反思习惯，明确下一步努力方向。总结在各个模块完成一轮练习后自动触发，也可手动在会话结束时查看。

**设计要点**
- 总结必须同时包含「亮点（做得好的）」和「待改进点」两个维度
- 呈现方式简洁，不超过一屏，避免信息过载
- 提供「继续学习」和「结束本次」两个出口
- 总结数据基于本次会话内的实际表现计算，不依赖历史均值

**验收场景**

| # | Given（前提）| When（操作）| Then（预期结果）|
|---|---|---|---|
| 7.1 | 用户完成一轮单词复习（≥5 张卡片）| 队列清空或手动结束 | 显示本轮单词总结：本次复习数量、评为「容易」的比例（亮点）、评为「困难」的单词列表（待改进）|
| 7.2 | 用户完成一个语法点的检验 | 所有检验题作答完成 | 显示语法总结：本次检验得分、答对的题目（亮点）、答错的题目及正确解析（待改进）|
| 7.3 | 用户完成一段影子跟读练习 | 录音评分完成 | 显示口语总结：本次相似度得分、与历史均值的对比（是否进步）、得分较低的句子标注（待改进）|
| 7.4 | 用户完成自由朗读练习 | 录音评分完成 | 显示口语总结：整体评分、发音准确的句子（亮点）、系统标注的发音需注意的词/句（待改进）|
| 7.5 | 用户完成当日所有造句练习 | 最后一题提交 | 显示写作总结：完成题数、平均得分、语法/用词全部正确的句子（亮点）、AI 标注的最常见错误类型（待改进）|
| 7.6 | 用户在任意模块查看总结 | 点击「查看本次详情」| 展开详细列表：每道题 / 每张卡片 / 每个句子的得分与反馈 |
| 7.7 | 用户在总结页 | 点击「继续学习」| 返回当前模块，可继续开始新一轮练习 |
| 7.8 | 用户在总结页 | 点击「今日全部完成」| 跳转到首页，显示今日整体学习成就（已完成模块数、总用时）|

## 边界情况

| 情景 | 系统行为 |
|---|---|
| **无网络** | 使用本地缓存的单词和课文数据，核心学习功能（复习、阅读）正常可用；进度本地暂存，恢复网络后自动同步 |
| **麦克风权限被拒** | 口语模块提示授权说明；允许用户在无录音情况下使用「只听不录」模式 |
| **单词队列为空** | 不展示空页面；提供「提前学习今日新词」或「浏览词库」选项 |
| **音频加载失败** | 课文文本仍可正常阅读；显示「音频加载失败，点击重试」按钮 |
| **AI 批改超时（>10s）** | 显示「批改中，请稍候」等待动画；超过 30 秒则提示「服务繁忙，请稍后重试」 |
| **多设备数据冲突** | 以最后更新时间戳为准合并；不弹窗打断用户学习流程 |
| **振り仮名渲染失败** | 降级展示纯日语文本，不显示假名；在页面底部显示提示 |
| **用户长时间无操作** | 保存当前学习进度到本地；下次打开从断点续学，不重置 |
| **练习数量过少（<3 个单元）** | 总结页仍然显示，但不显示「最常见错误」等统计项，改为提示「继续练习以获得更多分析」|
| **AI 批改失败导致总结数据不完整** | 跳过 AI 相关维度，仅展示本地可计算的统计项（完成数、得分等），并标注「部分数据暂不可用」|

## 功能性需求

### 模块 1：单词记忆

| ID | 需求描述 |
|---|---|
| FR-001 | 系统必须支持基于 **SM-2 间隔重复算法**的单词卡片调度，根据用户三级评分（容易 / 一般 / 困难）自动计算下次复习时间 |
| FR-002 | 单词卡片必须包含：日语单词（汉字/假名形式）、假名读音、词性、中文释义、至少 1 条例句（含中文翻译）|
| FR-003 | 系统必须显示今日待复习数量和新词学习进度，用户可提前学习新词 |
| FR-004 | 系统必须记录每个单词的复习历史、评分变化和掌握程度，支持查看统计 |
| FR-005 | 用户必须能够从课文和语法例句中**一键将生词加入**个人单词本 |
| FR-006 | 单词卡片必须支持**离线浏览**，已下载的单词数据在无网络时可正常使用 |
| FR-007 | 系统必须按 JLPT 等级（N5~N1）对词库分级，用户可按等级筛选或混合复习 |

### 模块 2：语法学习

| ID | 需求描述 |
|---|---|
| FR-008 | 系统必须支持按 JLPT 等级（N5~N1）浏览和筛选语法点，每个语法点包含：名称、意思、接续方式、至少 3 条例句（含翻译）|

| FR-014 | 系统必须维护每个用户的语法学习状态：未学 / 学习中 / 已掌握 |

### 模块 3：课文学习

| ID | 需求描述 |
|---|---|
| FR-015 | 系统必须支持日语课文的 **JLPT N5~N1 分级展示**，课文列表显示标题、字数、主题标签、等级 |

| FR-017 | 用户必须能点击任意单词查看**释义弹窗**，显示读音、词性、中文释义 |
| FR-018 | 系统必须支持课文音频的**分句播放**，播放时同步高亮当前句子 |
| FR-019 | 用户必须能切换课文的**中文翻译显示**（逐句对照模式） |

### 模块 4：口语练习

|---|---|
| FR-021 | 系统必须支持**影子跟读**子模式：选择音频材料，调节播放速度（0.5x~1.5x，步进 0.25x），文本同步高亮 |

| FR-029 | 输入练习必须支持**N5~N4 阶段的假名练习**（罗马字转假名）和 **N4+ 阶段的汉字练习**（给假名输入对应汉字）|

| FR-032 | 用户完成当日所有写作练习后，系统必须显示**今日写作总结**（完成题数、平均得分、常见错误类型）|

## 非功能性需求

### 架构解耦

| ID | 需求描述 |
|---|---|
| NFR-001 | **前后端分离**：后端以 RESTful API（或 GraphQL）对外提供所有数据服务，Web 端和未来的 iOS 端共用同一套 API，不允许在后端混入前端逻辑 |
| NFR-002 | **模块独立部署**：每个学习模块（单词 / 语法 / 课文 / 口语 / 写作）的后端服务必须可独立部署和扩展，模块间通过 API 调用，不直接共享数据库表 |
| NFR-003 | **内容与逻辑分离**：词库、语法点、课文内容存储于独立的内容数据库，通过管理后台维护，与用户学习数据严格隔离 |
| NFR-004 | **离线优先设计**：客户端必须实现本地数据缓存层，核心学习内容（已加载的单词、语法、课文）在离线状态下可访问，进度采用「本地优先 + 同步」策略 |
| NFR-005 | **平台适配层**：业务逻辑与平台 UI 解耦，确保 Web 版与 iOS 版共享相同的业务规则，仅在 UI 层有差异 |

## 关键实体

```
用户 (User)
  ├── id
  ├── email
  ├── 学习目标等级 (goal_level: N5~N1)
  ├── 注册时间 (created_at)
  └── 连续学习天数 (streak_days)

单词 (Word)
  ├── id
  ├── 日语写法 (kanji_form)
  ├── 假名读音 (reading)
  ├── 词性 (part_of_speech)
  ├── 中文释义 (meaning)
  ├── 例句列表 (examples[])
  └── JLPT 等级 (jlpt_level: N5~N1)

单词记录 (WordRecord)
  ├── user_id
  ├── word_id
  ├── 掌握程度 (mastery_level)
  ├── 下次复习时间 (next_review_at)
  ├── 历史评分列表 (review_history[])
  └── SM-2 算法参数 (ease_factor, interval)

语法点 (GrammarPoint)
  ├── id
  ├── 语法名称 (name, 如「〜てもいい」)
  ├── 中文释义 (meaning)
  ├── 接续方式 (conjunction_rule)
  ├── 使用场景说明 (usage_note)
  ├── 例句列表 (examples[])
  ├── 检验题列表 (quiz_questions[])
  └── JLPT 等级 (jlpt_level)

语法记录 (GrammarRecord)
  ├── user_id
  ├── grammar_point_id
  ├── 学习状态 (status: 未学/学习中/已掌握)
  ├── 下次复习时间 (next_review_at)
  └── 历史检验得分 (quiz_history[])

课文 (Lesson)
  ├── id
  ├── 标题 (title)
  ├── 正文（结构化含振り仮名标注）(content_with_furigana)
  ├── 中文翻译 (translation)
  ├── JLPT 等级 (jlpt_level)
  ├── 主题标签 (tags[])
  ├── 音频 URL (audio_url)
  └── 句子时间戳列表 (sentence_timestamps[])

口语练习记录 (SpeakingRecord)
  ├── user_id
  ├── 练习类型 (type: 影子跟读/自由朗读)
  ├── 材料 ID (material_id)
  ├── 练习时间 (practiced_at)
  ├── 相似度得分 (similarity_score: 0~100)
  └── 录音文件引用 (audio_ref)

写作记录 (WritingRecord)
  ├── user_id
  ├── 练习类型 (type: 输入练习/造句练习)
  ├── 题目内容 (question)
  ├── 用户输入 (user_answer)
  ├── AI 反馈 (ai_feedback)
  ├── 得分 (score)
  └── 练习时间 (practiced_at)

学习会话 (StudySession)
  ├── user_id
  ├── 模块类型 (module: 单词/语法/课文/口语/写作)
  ├── 时长 (duration_seconds)
  ├── 完成数量 (completed_count)
  └── 会话时间 (started_at)

练习总结 (SessionSummary)
  ├── user_id
  ├── 关联会话 (session_id)
  ├── 模块类型 (module: 单词/语法/口语/写作)
  ├── 亮点列表 (strengths[])          ← 本次做得好的项目
  ├── 待改进列表 (weaknesses[])       ← 需要重点关注的项目
  ├── 得分概要 (score_summary)        ← 本轮得分/正确率
  ├── 改进建议 (improvement_suggestions[])  ← 系统自动生成的 1~3 条建议
  └── 生成时间 (generated_at)
```

## 技术假设

| 假设项 | 内容 |
|---|---|
| **初始词库** | 系统内置至少 1000 个基础词汇（JLPT N5/N4 级别），后续逐步扩充至 N1 |
| **初始语法库** | 内置 N5+N4 完整语法点（约 100 条），后续迭代扩充 N3~N1 |
| **初始课文** | 内置 10 篇以上示例课文，覆盖 N5~N3 各 2~3 篇 |
| **音频来源** | 课文和跟读音频由专业母语者录制，存储于 CDN，不支持用户自制上传 |
| **口语评分方案** | 初期采用**整体音频相似度比对**（基于波形特征），不做单音节精细发音诊断 |
| **写作 AI 批改** | 调用大语言模型 API（如 Claude / GPT）对造句进行语法和用词批改 |
| **离线策略** | 用户学习数据存储于服务端，客户端本地缓存已学内容以支持离线使用 |
| **内容管理** | 初期内容由管理员后台维护，不提供用户自制课文上传功能 |
| **平台** | V1~V4 优先支持 Web 浏览器（桌面 + 移动响应式）；V5 迭代 iOS 原生 App |
| **后端语言** | Go（版本 ≥ 1.24），遵循项目宪法中的标准库优先原则 |
